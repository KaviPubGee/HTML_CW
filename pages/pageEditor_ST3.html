Feedback Page


Technical Description

The Feedback page uses a form with three fieldsets for personal details, feedback details, and ratings. It includes inputs like text, email, tel, file, and a textarea, with a datalist for topic selection. CSS uses flexbox for layout, with validation styling (red borders for invalid inputs, blue on focus) and hover effects on buttons and emojis (grayscale to color transition).


  • HTML Tags: <form>, <fieldset>, <legend>, <input>, <textarea>, <datalist>
  • CSS Techniques: Flexbox, validation pseudo-classes, hover effects, rgba borders

Accessibility

Labels are linked to inputs with for attributes, and title attributes guide validation errors. High-contrast text and keyboard-accessible inputs (radio, checkbox) ensure usability. Emojis lack aria-labels, which could be added for better screen reader support.

Team Page


Technical Description

The Team page displays four member cards in a flexbox grid. Each card has a relative-positioned image with an absolute-positioned details div that appears on hover (opacity transition). The layout is responsive with max-width and gap properties.


  • HTML Tags: <div>, <img>, <h2>, <ul>
  • CSS Techniques: Flexbox, absolute positioning, opacity transitions, object-fit

Accessibility

Images have descriptive alt text, and cards are keyboard-focusable with tabindex="0". High-contrast text aids readability, but hover-only details could use focus styles for better keyboard accessibility.

Content Page


Technical Description

The Content page features a sticky sidebar with anchor links to sections on coral reefs, coastal habitats, food webs, and human dependence. Content is organized in flexbox cards with responsive images. A fixed "Go to Top" button enables smooth scrolling.


  • HTML Tags: <aside>, <section>, <ul>, <img>, <a>
  • CSS Techniques: Sticky positioning, flexbox, smooth scrolling, responsive images

Accessibility

Descriptive sidebar links and alt text for images support screen readers. A clear heading structure and keyboard-accessible "Go to Top" button (with aria-label) enhance navigation. High-contrast colors ensure readability.

Challenges and Lessons Learned


On the Feedback page, preventing premature red borders on invalid inputs was tricky. Using :invalid:not(:placeholder-shown) fixed this. The Team page’s hover effects weren’t keyboard-accessible, so I added tabindex="0". The Content page’s sticky sidebar needed scroll-margin-top to avoid header overlap.


Lesson: Accessibility requires testing across mouse, keyboard, and screen reader interactions to ensure inclusivity.


I learned the importance of semantic HTML and validation early in development to catch issues before deployment.


Compliance


My pages use semantic HTML (<header>, <main>, <section>) for clear structure. All images include descriptive alt text for screen readers.


Color contrast and font sizing were checked for readability. Navigation is keyboard-accessible with smooth scrolling for user experience.


CSS and HTML passed W3C validation. All external resources are licensed or royalty-free, aligning with JANET guidelines for web publishing.
