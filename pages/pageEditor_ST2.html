Home Page


Technical Description

The home page uses a minimalist layout with centered text element and a button to introduce the site to the user. It also consists of our mission, and finally a beautifully layed out set of cards introducing the user to each subtopic we prepared.


  • HTML Tags: <nav>, <h1>, <a>, <main>, <section>
  • CSS Techniques: Scroll animation, Hover effects, Blur effects, background opacity

Accessibility

The home page includes some alt texts for all the images for screen readers, as well as clear headings with good color contrasts, keyboard-navigable buttons.

Table Page


Technical Description

This page uses tables to display the ocean pollution levels and consists of alternating row colors.


  • HTML Tags: <table>, <thead>, <tbody>, <tr>, <th>, <td>
  • CSS: border-collapse, box-shadow, hover states, max-width, responsive font sizing

Accessibility

Table headers use <th> for screen reader accessibility. The table has a contrast for readability and the used structure helps screen readers can navigate by column and row.

Content Page


Technical Description

This page is a scrollable one-pager focused on the topic of Ocean Pollution. It features a fixed vertical sidebar with links to key sections, such as: Introduction, Causes, Effects, and Solutions. The main content is organized using flexbox and placed inside glass-style cards with blur effects.


  • HTML Tags: <aside>, <section>, <nav>, <ul>, <h2>
  • CSS: position: fixed, flexbox layout, backdrop-filter blur, semantic sectioning, scroll-margin-top

Accessibility

The sidebar links use descriptive links for screen readers. The content uses high-contrast colors, and large font sizes for readability. All images have alt text and are aligned to not obstruct reading flow.

Challenges and Lessons Learned


For the Content page, managing a fixed sidebar without overlapping the footer was tricky. I experimented with positioning and grid layouts to keep the sidebar accessible without breaking the layout. This taught me how important layout planning and scroll margin are. Another challenge I faced was aligning the logo in the navigation bar, this was a easy fix with the use of <div> tags. This gave a me a better understanding on blocklevel and inline elements.


Lesson: Proper clear design and testing are crucial for accessibility and the best possible user experience.


I also learned the importance of using animations to make the website feel more smoother all while not sacrificing the performance .


Compliance


My pages follow HTML5 standards with semantic tags like <header>, <nav>, <main>, and <section> for accessibility and clear structure. All images include descriptive alt text for screen readers.


Color contrast and font sizing were checked to ensure readability. Navigation is consistent and keyboard-accessible, with smooth scrolling to improve user experience.


CSS and HTML were validated using the W3C validation tools. All external resources are licensed for use or royalty-free. The design is in line with JANET guidelines and best practices for web publishing.
