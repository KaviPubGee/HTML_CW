üöß Challenges & Lessons Learned

I faced issues with layout collapsing in smaller viewports on the Volunteer page. I resolved it using media queries and testing across different screen sizes. I also refined the internal linking and scrolling behavior using anchor links and sticky nav to improve user flow.


Lesson: Testing across devices is essential to ensure responsive design. Using semantic HTML and CSS Flexbox can greatly enhance layout flexibility.


Lesson: Proper responsive design and testing are crucial for accessibility and user experience.


I also learned the importance of using semantic HTML elements for better accessibility and SEO. The use of Flexbox allowed for a more fluid layout that adapts to different screen sizes without breaking.


Additionally, I gained experience in using CSS animations and transitions to enhance the visual appeal of the pages while maintaining performance.

‚ôø Accessibility

All pages include alt text for images, semantic HTML structure for screen readers, keyboard-accessible navigation, and color contrast consideration to ensure readability. The content page uses high-contrast buttons and avoids text embedded in images.


The splash screen uses a preloader to ensure content is fully loaded before display, improving user experience. All links are descriptive and open in new tabs where necessary to avoid losing context.


The site is tested with screen readers and keyboard navigation to ensure all interactive elements are accessible. The use of ARIA roles and properties is implemented where necessary to enhance accessibility.


The content page includes a scroll-to-top button for easier navigation on long pages, ensuring users can quickly return to the top without excessive scrolling.
