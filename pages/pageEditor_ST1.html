Volunteer Page


Technical Description

This page uses glassmorphism and card layouts to present volunteer profiles and user reviews. Responsive Flexbox and interactive hover states enhance UX.

  • HTML Tags: <section>, <article>, <button>
  • CSS: Flexbox, media queries, hover transitions, box-shadow, frosted glass effect

Accessibility

All pages include alt text for images, semantic HTML structure for screen readers, keyboard-accessible navigation, and color contrast consideration to ensure readability. The content page uses high-contrast buttons and avoids text embedded in images.

Technical Description

The editor page serves as a personalized portfolio for Student 1, showcasing responsibilities, technical breakdowns, accessibility features, and more. It follows a semi-independent theme with frosted glass styling and organized content blocks using semantic HTML and a logical heading structure.

Challenges and Lessons Learned


I faced issues with layout collapsing in smaller viewports on the Volunteer page. I resolved it using media queries and testing across different screen sizes. I also refined the internal linking and scrolling behavior using anchor links and sticky nav to improve user flow.


Lesson: Testing across devices is essential to ensure responsive design. Using semantic HTML and CSS Flexbox can greatly enhance layout flexibility.


Lesson: Proper responsive design and testing are crucial for accessibility and user experience.


I also learned the importance of using semantic HTML elements for better accessibility and SEO. The use of Flexbox allowed for a more fluid layout that adapts to different screen sizes without breaking.


Additionally, I gained experience in using CSS animations and transitions to enhance the visual appeal of the pages while maintaining performance.

Compliance


Pages were built with valid HTML5/CSS3, tested through W3C validators. All external resources are credited. No copyrighted material is used without permission. Links include descriptive text and open in new tabs where needed to meet Janet web regulations.


The site follows best practices for web authoring and accessibility guidelines, ensuring a user-friendly experience for all visitors. Regular audits are conducted to maintain compliance with web standards and accessibility requirements.


The site is designed to be responsive, ensuring it works well on various devices and screen sizes. All pages are linked correctly, and the navigation structure is clear and intuitive.
